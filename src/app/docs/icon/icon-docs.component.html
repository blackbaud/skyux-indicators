Icons represent actions, content, and ideas, and they help define the visual identity of SKY UX applications. The icon component displays Font Awesome icons or custom SKY UX icons.

Icons deliberately facilitate user comprehension and drive focus to actions or sections, and they follow these principles:

  • Global: Use established, universally-known icons in a purposeful manner wherever possible

  • Supplemental: Icons should be paired with text wherever possible; avoid using icons solely for decoration or visual interest

  • Specific: Icons should have a single meaning, and variations should be limited: A single icon should represent a common idea or subject and should not be abstract.

SKY UX uses simple, glyphic, mono-color icons to facilitate, guide, and inform users. Icons are designed in Blackbaud's brand personality, reflecting a human, visionary approach, and are built on the SKY UX's modern theme's visual language.

Icons are always a solid, monochromatic color and must meet the WCAG contrast ratio of 3:1. An icon's color should be reflective of the component to which it is tied. For more information, view the SKY UX color guidelines for modern theme.

SKY UX icons have two different style variations: outlined and filled. Use the outlined variation for all icons displayed within your experiences. The filled variation is only used to indicate state changes on a given component as described in its documentation.

Icons usually inherit their size and color from parent elements, particularly when you use them inline. SKY UX icons are typically 16px to match standard body copy. Icons of varying size can be used depending on its parent's component specifications. 

The standard set of SKY UX icons are selected with global considerations with the goal to ensure that they are understood by international users of varying cultures and backgrounds. 

Icons in SKY UX should be used in a way that meet WCAG contrast standards. Icons must meet a 3:1 contrast ratio with the background on which they're applied. 

Icons should have meaningful text labels where possible.

Avoid using icons solely for visual interest or decoration.

Do not use icons with established meanings for different purposes.

To request an icon not currently available in the SKY UX icon library, please reach out to the @skyux-team design team.

SKY UX also provides a set of custom icons.
